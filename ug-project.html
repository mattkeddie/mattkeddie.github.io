Educational Web Security Site

UNDERGRADUATE PROJECT

A concept of a site to be used extra curricularly for students who wish to deepend their understanding of Information Security topics. There is a seemingly apparent skill gap in the sector, however there is also the problem that if a school tried to push this as some sort of mandatory part of a syllabus, it wouldn't appeal to a lot of people. Hence, the premise of this site was to make it somewhat extra-curricular for those who are interested to dive deeper into the world of Information Security and enrich their knowledge.

I do have a report written as part of this project too which goes even more into depth about how it was created, however this will not be distributed unless there is a very good reason for it. The same also goes for access to the GitHub Repository where the code is stored.

Showcase

Below are some screenshots of every page of the site along with a description of what the goal of each page is and what exactly can be done, as well as discussing potential limitations of the platform. As this is only the concept of the platform, this has been designed around using a LocalHost for hosting the platform, meaning that this is not available for public use. I do also apologise if the screenshots are hard to read, I'm not entirely sure what has caused them to appear weirdly on some resolutions.

Login Page

The first page that will be displayed to all users. You cannot bypass this if you are not logged in, which would be useless anyway as you would have no tasks assigned to you! Every page requires a user to be logged in and if there is no current user then they will be redirected to here. There is also the option for admins to log in.

Student Task List

When a user logs in, they will be first greeted by a screen containing a list of all their tasks that are both incomplete and complete. These tasks are user specific as to what they have set to them.

With the aforementioned list of tasks, it is entirely possible to filter the list which is done by AJAX. As demonstrated by the below screenshot, you can type part of a category name in and upon clicking search it will display only the relevant category of tasks. When this is blank it shows everything.

Task Detailed View

If you click view task on any task on the list, you will see a page of the same format as this. From here you can either click to mark the task as complete, or you can view the task.

Task

If you click link to task, you will have a new tab open with the task inside it.

If you depending on whether you click the correct or incorrect answer, you will get the relevant feedback.

Mark Task as Complete

When you finish the task you will wish to mark it as complete. This is as simple as clicking a button, ticking a tick box and then clicking another button!

You will then be redirected back to the task list page where the task you just marked as complete will now appear in green!

Resource List

This platform is supposed to be made as good as the users make it. There is a space for users to post resources they find useful. Of course, this section can be moderated and inappropriate posts cleared as well as altering user permissions.

Resource Detailed View

Similar to the task list, clicking on the resource name will take you to a detailed page showing the topic and a brief description of the resource.

External Resource

Clicking on the link to the resource will redirect you to what is likely to be an external site depending on the resource. To avoid interruption with the session on the platform, it will open in a new tab.

Admin Login

As mentioned in the login section of this showcase, admins have their own separate login. From this point on we will be focussing only on the admin section of the site. Only users that are a "superuser" may login through this portal.

Admin View

Admins can control essentially any data held in the database through Django's inbuilt admin view. This has been slightly customised but not that much, mainly just the page title. Here is where they can select what it is they would like to edit.

Admins can see any user that is signed up on the platform.

Along with access to any details that could possibly need viewing or amending.

This also applies to tasks where they can view all tasks on the database.

They can also edit an existing task or assign a new task.

Lastly, this also applies for resources.

Hopefully this screen will never need to be used by an admin, but there is a big red button here which can delete any inappropriate posts if necessary.
